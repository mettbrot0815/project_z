# Z (1996) Godot 4.6 Recreation Implementation Plan

## Project Overview
Recreating the classic 1996 Bitmap Brothers real-time strategy game **Z** as a modern, faithful, production-ready cross-platform title using Godot 4.6.

---

## Success Criteria
- 95%+ faithful recreation of original gameplay mechanics
- Full single-player campaign (20 levels across 5 planets)
- Multiplayer support up to 4 players
- Exact original balance, timing, and feel
- Production-ready quality with modern QoL improvements

## Critical Technical Requirements
### Godot 4.6 Specific Optimizations
1. Use `NavigationServer2D` with obstacle avoidance for unit pathing
2. Enable `multi_threaded` physics for 100+ units simulation
3. Use `Sprite2D` with texture atlases for performance
4. Implement object pooling for projectiles / effects
5. Use `RPC` with authority mode for network sync

### Original Mechanics Preservation
- **NO** resource gathering or base building
- Production speed **exponentially** scales with territory count
- Factory timers continue running when captured mid-build
- Unmanned vehicles are immediately claimable by any player
- Robots have 5 tiers of intelligence dictating autonomous behavior
- Exploding turrets fly and cause area damage when landing

## Risk Assessment
| Risk | Severity | Mitigation |
|---|---|---|
| Original timing breaks on modern hardware | HIGH | Use fixed timestep `_physics_process` for all game logic |
| Pathfinding performance with 100+ units | MEDIUM | Enable navigation server threading, batch path requests |
| Network desync in multiplayer | HIGH | Authoritative host, deterministic simulation, delta sync |
| Balance deviates from original | MEDIUM | Automated timing tests, reference playthroughs |
