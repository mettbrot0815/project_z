# Implementation Plan: Complete Z (1996) Recreation in Godot

## Project Overview
Based on codebase analysis, the Godot recreation of Z (1996) is approximately 80-85% complete. Core systems (territory control, production scaling, combat, vehicle mechanics) are implemented, but several key features are missing or incomplete to achieve 95%+ fidelity to the original game as specified in the project summary.

## Current State Assessment
- **Implemented**: Territory control, factory production with scaling, combat system, vehicle driver sniping, stationary guns, victory conditions, unit roster (13 types), level loading framework, UI controls.
- **Partially Implemented**: Unit AI (shooting complete, movement incomplete for 8/13 units), campaign (only Level 1).
- **Missing**: Full unit movement AI, remaining 19 campaign levels, multiplayer networking, assets (sprites/sounds), visual effects.

## Goal
Achieve production-ready status with all core mechanics faithful to original Z, including 20-level campaign, multiplayer skirmish, and QoL features while preserving retro feel.

## Implementation Phases

### Phase 1: Complete Unit AI Movement (Priority: High)
**Objective**: Implement autonomous movement AI for all intelligence levels to match original behaviors.
**Estimated Time**: 2-3 days
**Files to Modify**: `scripts/core/unit_base.gd`, `scripts/units/*.gd` (grunt, psycho, sniper, vehicles)

**Steps**:
1. **Intelligence Level 0 (Grunt)**: Implement flag-rushing behavior
   - Add `_find_nearest_flag()` in unit_base.gd
   - Modify grunt.gd to prioritize moving toward unclaimed/captured flags
   - Add pathfinding integration with NavigationServer2D

2. **Intelligence Level 1 (Psycho)**: Implement aggressive nearest-enemy pursuit
   - Add `_find_nearest_enemy()` in unit_base.gd
   - Update psycho.gd to continuously target closest enemy unit
   - Balance movement speed vs. firing rate for optimal aggression

3. **Intelligence Level 2 (Tough)**: Complete vehicle-targeting movement
   - Enhance existing vehicle-finding logic in tough.gd
   - Add pathfinding to approach vehicles while avoiding threats
   - Integrate with existing threat detection

4. **Intelligence Level 3 (Sniper)**: Implement driver-priority movement
   - Add `_find_nearest_driver()` in unit_base.gd
   - Update sniper.gd to move toward vehicles with drivers
   - Combine with existing shooting priority for drivers

5. **Vehicle AI**: Implement intelligence-based vehicle behaviors
   - jeep.gd: Low intelligence, basic flag rush
   - light_tank.gd: Balanced, target threats
   - medium_tank.gd: Prioritize enemy vehicles
   - heavy_tank.gd: Defensive positioning
   - apc.gd: Transport troops to flags
   - crane.gd: Repair damaged units/buildings
   - missile_launcher.gd: Seek high-value targets

6. **Testing**: Verify AI behaviors match original unit personalities
   - Test in Level 1 with various unit combinations
   - Ensure no infinite loops or performance issues

### Phase 2: Complete Campaign System (Priority: High)
**Objective**: Add all 20 campaign levels with progressive difficulty and original cutscenes.
**Estimated Time**: 1-2 weeks
**Files to Modify**: `data/levels.json`, `scripts/campaign/level_loader.gd`, new cutscene system

**Steps**:
1. **Level Data Creation**: Add JSON definitions for Levels 2-20
   - Include territory layouts, factory placements, starting units
   - Implement AI production cheats for CPU side in later levels
   - Add special level features (bridges, destructible terrain)

2. **Planet Themes**: Implement visual theming for all 5 planets
   - Desert (existing), Volcanic, Arctic, Jungle, City
   - Background colors, terrain sprites, atmospheric effects

3. **Cutscene System**: Add cutscenes for Commander Zod, Brad & Allan
   - Implement rocket-fuel humor endings
   - Add mission briefings and victory/defeat screens
   - Integrate with level progression

4. **Difficulty Progression**: Balance AI advantages
   - Starting turrets for CPU in later levels
   - Production multipliers and cheats
   - Unit composition scaling

5. **Level Progression**: Add campaign flow
   - Unlock next level on victory
   - Save/load campaign progress
   - Statistics tracking (units lost, time taken)

### Phase 3: Multiplayer Implementation (Priority: Medium)
**Objective**: Add hotseat and networked multiplayer for up to 4 players.
**Estimated Time**: 1 week
**Files to Create**: `scripts/multiplayer/`, new scenes

**Steps**:
1. **Hotseat Mode**: Implement local multiplayer
   - Turn-based or simultaneous control
   - Player switching UI
   - Shared camera or split-screen

2. **Networked Multiplayer**: Use Godot's MultiplayerAPI
   - Client-server architecture
   - Synchronize unit positions, orders, territory ownership
   - RPC calls for production and combat events

3. **Lobby System**: Add matchmaking
   - Map selection from 16 MP maps
   - Player ready states
   - Connection handling and error recovery

4. **Synchronization**: Ensure deterministic gameplay
   - Fixed timestep for physics
   - State reconciliation for late joins
   - Anti-cheat measures (server authority)

### Phase 4: Assets and Visual Polish (Priority: Medium)
**Objective**: Add retro pixel-art assets, audio, and effects to match original feel.
**Estimated Time**: 1 week
**Files to Add**: `assets/` directory with subfolders

**Steps**:
1. **Sprite Assets**: Create or source 32x64px pixel art
   - Unit sprites for all 13 types (animations: walk, shoot, damage states)
   - Building sprites (flags, factories, forts)
   - Terrain tiles, explosions, debris
   - UI elements matching original sidebar

2. **Audio Assets**: Add voice lines and SFX
   - Unit barks (humorous lines like "You are so crap")
   - Combat sounds (gunfire, explosions)
   - UI feedback (button clicks, production complete)
   - Background music per planet

3. **Particle Effects**: Implement visual explosions
   - Flying turret physics on vehicle death
   - Debris scattering, smoke trails
   - Splash damage indicators

4. **Animation System**: Add sprite animations
   - Unit movement, turret rotation
   - Damage transitions (smoke → oil → fire)
   - Explosion sequences

### Phase 5: Testing, Balancing, and QoL (Priority: High)
**Objective**: Ensure fidelity to original balance, add modern conveniences.
**Estimated Time**: 3-5 days
**Files to Modify**: Various, add new systems

**Steps**:
1. **Balance Testing**: Playtest against original
   - Verify production curves, unit stats, AI behaviors
   - Adjust for modern hardware (ensure no timing breaks)
   - Test all 20 levels for difficulty progression

2. **QoL Features**: Add without changing core balance
   - Save/load system for campaigns
   - Modern resolution support (widescreen, scaling)
   - Improved pathfinding (A* with terrain costs)
   - Accessibility options (colorblind support, key remapping)

3. **Performance Optimization**: Ensure smooth 60fps
   - Optimize pathfinding for large unit counts
   - Efficient collision detection
   - Memory management for assets

4. **Bug Fixes**: Address edge cases
   - Bridge repair mechanics
   - Terrain destruction pathfinding updates
   - Grenade/terrain interaction

### Phase 6: Deployment and Documentation (Priority: Low)
**Objective**: Prepare for release on itch.io/Steam.
**Estimated Time**: 2-3 days

**Steps**:
1. **Build Configuration**: Export templates for multiple platforms
2. **Documentation**: Update README with full feature list
3. **Credits**: Attribute original Bitmap Brothers
4. **Bonus Content**: Add level editor for community maps

## Success Criteria
- All 20 campaign levels playable with original difficulty curve
- Multiplayer functional for 2-4 players
- 95%+ visual/audio fidelity to original
- Core balance unchanged (no resource gathering, instant territory capture)
- Production-ready with no critical bugs

## Risk Mitigation
- Regular testing after each phase to catch regressions
- Modular implementation to allow iterative development
- Fallback to placeholder assets if custom art delayed
- Use existing framework (NavigationServer2D, MultiplayerAPI) for complex systems

## Timeline Estimate
- Total: 4-6 weeks
- Phase 1-2: 2-3 weeks (critical path)
- Phase 3-6: 2-3 weeks (parallelizable)

This plan prioritizes core gameplay completion first, then expands to full feature set while maintaining the lightning-fast, tactical feel of the original Z.
